# Sign Transaction

Signs a prepared transaction with the provided signature and public key.

<div className="bg-yellow-100 border-l-4 border-yellow-500 text-yellow-700 p-4 mb-4">
  <strong>⚠️ Development Only:</strong> This endpoint is primarily for development and testing purposes.
</div>

## Endpoint

```
POST /sign-tx
```

## Request Body

```json
{
  "payload": {
    "title": "string",
    "fraction_count": 1000,
    "description": "string",
    "tags": ["tag1", "tag2"],
    "metadata": {},
    "requirements": {},
    "lockup_options": {},
    "feed_url": "string"
  },
  "signature": "string",
  "public_key": "string",
  "encoded_transaction_body": "string"
}
```

### Parameters

- **payload** (required): The mint request payload (same format as mint creation)
  - **title**: Title of the tokenized asset
  - **fraction_count**: Number of token fractions to create
  - **description**: Description of the asset
  - **tags**: Array of tags
  - **metadata**: Additional metadata object
  - **requirements**: Requirements for token ownership
  - **lockup_options**: Token lockup configuration
  - **feed_url**: URL to the asset's data feed
- **signature** (required): Hex-encoded signature of the payload
- **public_key** (required): Hex-encoded public key for verification
- **encoded_transaction_body** (required): Base64 or hex-encoded transaction body

## Response

### Success (200 OK)

The response format depends on the implementation and what the signed transaction produces.

### Error Responses

- **400 Bad Request**: Invalid JSON or validation failed
- **500 Internal Server Error**: Server error during signing

## Example

### cURL

```bash
curl -X POST "http://localhost:8080/sign-tx" \
  -H "Content-Type: application/json" \
  -d '{
    "payload": {
      "title": "Sample Token",
      "fraction_count": 1000,
      "description": "A sample token for testing",
      "tags": ["sample", "test"],
      "metadata": {},
      "requirements": {},
      "lockup_options": {},
      "feed_url": "https://example.com/feed"
    },
    "signature": "304502210...",
    "public_key": "03a1b2c3d4e5f6...",
    "encoded_transaction_body": "0100000001..."
  }'
```

### Python

```python
import requests

payload = {
    "payload": {
        "title": "Sample Token",
        "fraction_count": 1000,
        "description": "A sample token for testing",
        "tags": ["sample", "test"],
        "metadata": {},
        "requirements": {},
        "lockup_options": {},
        "feed_url": "https://example.com/feed"
    },
    "signature": "304502210...",
    "public_key": "03a1b2c3d4e5f6...",
    "encoded_transaction_body": "0100000001..."

response = requests.post('http://localhost:8080/sign-tx', json=payload)
result = response.json()
print("Transaction signed:", result)
```

## Use Cases

- **Development Testing**: Sign prepared transactions in test environments
- **Integration Testing**: Automated signing of test transactions
- **Debugging**: Test transaction signing without full wallet integration
- **Prototyping**: Quick transaction signing for proof-of-concept development

## Notes

- This endpoint is primarily for development and testing
- The transaction body should be properly encoded (base64 or hex)
- The signature must be valid for the provided payload and public key
- The payload format matches the mint creation request structure
- In production, transaction signing should typically be done client-side for security
- The encoded transaction body represents the raw transaction data to be signed
